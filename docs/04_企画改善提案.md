# 企画改善提案

## エグゼクティブサマリー

1. **試合後の打席入力を「5分で完了」できるUXに最適化**する。現状の1打席ずつフォーム入力方式は草野球プレイヤーの「試合後にビール片手で」という利用シーンに対して重すぎる。
2. **因縁カードを「SNSで見せたくなる」品質に引き上げ**、バイラル要素と招待導線を組み込む。現状のカードデザインは情報提示としては成立しているが、SNS上で目を引く「個性」と「文脈」が不足している。
3. **相手チームへの自然な招待導線を設計**し、ネットワーク効果を加速させる。因縁カードの共有が「アプリのインストール」につながるフローが未整備。

---

## 1. ユーザージャーニー分析

### 1.1 現状のユーザーフロー（登録〜初回因縁カード生成）

```
アプリ起動 → メール/PW登録 (SignUpPage: 3フィールド)
  → ホーム画面 (空の状態)
  → チーム作成 (CreateTeamPage: チーム名+エリア)
  → チーム詳細 → 試合を追加 (CreateGamePage: 日付+球場+対戦チーム)
  → 試合詳細 → 打席を追加 × N回 (PlateAppearanceInputPage: 1打席ずつ)
  → 試合確定
  → ホーム → 因縁を見る → 因縁一覧 → 因縁詳細 → カード生成 → 共有
```

**所要ステップ数**: 最低でも 8画面遷移 + N回の打席入力（各打席に4-5タップ）

**KPI「3分以内に因縁カード生成」は未達成**。理由:
- 打席入力のたびに「バッター所属チーム選択 → バッター選択 → ピッチャー所属チーム選択 → ピッチャー選択 → 結果タイプ → 結果詳細」という6ステップが必要
- 初回は選手が未登録のため、全員「新規作成」が必要
- 因縁カードへの導線が「ホーム → 因縁一覧 → 因縁詳細 → カード生成」と深い

### 1.2 課題の構造

| フェーズ | 課題 | 影響度 |
|----------|------|--------|
| 初回登録 | ソーシャルログインがなく、メール登録が面倒 | 高 |
| 初回セットアップ | チーム作成 → 選手登録の導線が長い | 高 |
| 試合入力 | 1打席ずつの入力が重い（試合20打席で20回フォーム入力） | 最高 |
| 因縁体験 | 因縁カードへの到達が深い | 中 |
| 共有・招待 | カード共有はあるが、アプリへの招待導線がない | 高 |
| 継続利用 | 次の試合を入力するモチベーション設計がない | 中 |

### 1.3 改善後の理想フロー

```
アプリ起動 → Appleでサインアップ (1タップ)
  → オンボーディング: チーム作成 or 参加 (招待コード入力)
  → チーム詳細 → 試合を作成
  → 連続打席入力モード (スワイプ/タップで高速入力)
  → 試合確定 → 自動で「注目の因縁カード」を生成・表示
  → 1タップで共有
```

## 2. 機能改善提案（優先度順）

### P0（即実装すべき）

#### 2.1 打席入力の高速化「クイック入力モード」

**現状の問題点**:
- `PlateAppearanceInputPage` は1打席ごとにフォームを開いて入力 → 保存 → 戻る → また追加、を繰り返す
- 1打席あたりの操作: チーム選択(2タップ) + 選手選択(2タップ) + 結果(2タップ) + 保存 = 最低6タップ + 画面遷移2回
- 20打席の試合だと最低120タップ + 40回の画面遷移

**改善案**:
- **連続入力モード**: 打席登録後に画面をリセットせず、ピッチャーを維持したまま次のバッター入力に移行
- **「前の打席を複製」ボタン**: 同じピッチャー vs 異なるバッターのパターンが多いため
- **スコアブック風一括入力UI（将来）**: 試合全体をまとめて入力できるスプレッドシート的UI

**なぜ刺さるか**: 草野球の試合後は疲れている。仲間との飲み会中に「サクッと入力できる」ことが継続利用の鍵。5分を超えると「また今度でいいや」になり、永遠に入力されない。

#### 2.2 因縁カードのデザイン・コンテンツ強化

**現状の問題点**:
- `RivalryCardWidget` のカード（600x400固定サイズ）は情報提示としては機能するが、SNS映えする要素が弱い
- コメントのバリエーションが少ない（5パターンのみ: 天敵認定/圧倒的な打棒!/好相性の打者/互角の勝負/苦手な投手）
- 「VS」の対決感が文字のみで視覚的インパクトが弱い
- 背景のグラデーション+斜線パターンは無機質

**改善案**:
- **コメントパターンの拡充**: 打率だけでなく、対戦数・HR数・三振数の組み合わせで30パターン以上に
- **対戦回数バッジ**: 「宿敵度レベル」を星や炎のアイコンで視覚化（例: 10打席以上 = 炎3つ、20打席以上 = 炎5つ）
- **因縁ストーリー**: 「3試合連続ヒット中」「最近5打席で4三振」のようなトレンド情報をカードに追加
- **アスペクト比の調整**: 600x400 → 1080x1350（Instagram推奨）/ 1200x630（Twitter/LINE OGP推奨）の選択
- **カードテーマ選択**: 「炎の対決」「氷の因縁」「名勝負」などのテーマカラー/テンプレートを用意

**なぜ刺さるか**: カードの「面白さ」がシェアの原動力。数字だけでは「へー」で終わるが、「天敵認定 炎炎炎炎炎」は笑いが起きる。草野球の仲間内LINEグループで「見て見て！」と送りたくなる要素が必要。

#### 2.3 試合確定後の自動因縁カード提案

**現状の問題点**:
- 試合確定後、因縁カードへの導線が `GameDetailPage` → `ホーム` → `因縁一覧` → `詳細` → `カード生成` と5ステップ必要
- ユーザーが「因縁カードを作れる」ことに気づかない可能性がある

**改善案**:
- 試合確定時に「注目の因縁」をピックアップし、ダイアログまたはボトムシートで表示
  - 例: 「田中 vs 鈴木: 打率.500に上昇！因縁カードを作る？」
- 試合詳細ページの確定後に「因縁カードを作成」CTAを表示

**なぜ刺さるか**: 入力完了直後が最も「結果を見せたい」モチベーションが高い瞬間。その場でカードを作れれば、共有率が飛躍的に向上する。

#### 2.4 ホーム画面の「因縁ハイライト」セクション

**現状の問題点**:
- `HomePage` は「自分のチーム」「直近の試合」の2セクション + 「因縁を見る」ボタンのみ
- 因縁データ（このアプリのコアバリュー）がホーム画面で一切表示されていない
- ホーム画面にアプリの「楽しさ」が感じられない

**改善案**:
- ホーム画面に「注目の因縁」カルーセルを追加
  - 打率が高い/低い対戦、最近対戦が増えている組み合わせをピックアップ
  - タップで因縁詳細 → カード生成へ直行
- 「今月のハイライト」（最多HR、最高打率の因縁ペアなど）

**なぜ刺さるか**: アプリを開くたびに「自分の因縁」が目に入ることで、「もっと試合を入力しよう」という動機づけになる。現状のホーム画面は「管理ツール」の印象で、「楽しいアプリ」の印象がない。

### P1（MVP強化）

#### 2.5 オンボーディングフローの導入

**現状の問題点**:
- 登録後にホーム画面に到達しても「何をすべきか」が分からない
- `HomePage` のチーム未参加時は「チームに参加していません」＋「チームを作成する」ボタンのみ
- 初回ユーザーへのガイダンスがゼロ

**改善案**:
- 3ステップのオンボーディング
  1. 「チームを作成 or 招待コードで参加」の選択画面
  2. チーム作成/参加完了後、「最初の試合を記録しましょう」への誘導
  3. 試合入力完了後、「因縁カードが生成できます！」の祝い画面
- プログレスバーで「あと1ステップで因縁カードが作れます」表示

**なぜ刺さるか**: 初回体験で「因縁カード」の面白さを体験できないと、ユーザーはアプリの価値を理解する前に離脱する。

#### 2.6 ソーシャルログイン（Apple / Google）

**現状の問題点**:
- `LoginPage` / `SignUpPage` はメール/パスワードのみ
- 草野球プレイヤー（30-40代男性）はメール登録を面倒に感じる層が多い

**改善案**:
- Apple Sign In / Google Sign In の追加（基本設計書にも「優先度高：後続」と記載あり）
- 登録フォームの表示名はソーシャルログインの情報から自動取得

**なぜ刺さるか**: 「試合後の飲み会で友達に勧められてその場でインストール」というシーンで、メール入力は致命的なハードル。Appleサインインなら3秒で完了。

#### 2.7 因縁カードからのアプリ招待導線

**現状の問題点**:
- `CardPreviewPage` の共有ボタンは画像を共有するのみ
- 共有された側が「このアプリ何？」となっても、ダウンロードへの導線がない
- ウォーターマーク「base_match」は入っているが、検索しないとたどり着けない

**改善案**:
- カード画像にQRコード / App Storeリンクを埋め込み
- 共有テキストに「base_matchで因縁を記録しよう！ [ストアURL]」を自動付与
- カード内に「あなたとの因縁をもっと見る」の招待メッセージ追加
- 将来: Dynamic Links でカード → アプリ内の因縁詳細に直接遷移

**なぜ刺さるか**: 草野球は必ず「相手チーム」がいる。対戦相手に因縁カードを送りつける行為自体がアプリの宣伝になる。「お前、俺との因縁はこうなってるぞ」と送れば、相手は確実に興味を持つ。

#### 2.8 チーム選手の事前登録

**現状の問題点**:
- `PlateAppearanceInputPage` で選手を選ぶとき、初回は選手が登録されていない
- 毎回「新規作成」で選手名を手入力する必要がある
- チーム詳細にメンバー一覧はあるが、選手（players）の管理UIがない

**改善案**:
- チーム詳細画面に「選手登録」セクションを追加
- 試合前にスターティングメンバーを事前登録しておけるUIを提供
- チームメンバー（user）と選手（player）の自動紐づけ

**なぜ刺さるか**: 試合後に選手名を入力する作業は単純に面倒。事前に選手リストがあれば、打席入力は「タップで選択」するだけになり、入力速度が大幅に向上する。

## 3. UX/UI改善提案

### 3.1 ホーム画面 (`home_page.dart`)

**現状**: チーム一覧 + 直近試合 + 因縁ボタン。情報が少なく、管理ツール的な印象。

**改善案**:
- **ヘッダーを「因縁ハイライト」に変更**: 「ようこそ」の挨拶文より、直近の注目因縁をカルーセルで表示
- **BottomNavigationBar の導入**: 現状は因縁一覧へのアクセスが「ボタンタップ → 別画面」だが、ホーム/試合/因縁/プロフィールのタブバーにすべき
- **FAB (Floating Action Button)**: 「試合を記録する」ボタンを常に画面右下に配置。最もよく使うアクションへの即座のアクセス
- **空状態のデザイン改善**: 「チームに参加していません」のカードが素っ気ない。イラストと説明文を追加し、「2分で始められます」のメッセージを添える

### 3.2 打席入力画面 (`plate_appearance_input_page.dart`)

**現状**: フォーム形式。1打席ごとに全フィールドを入力。スクロールが長い。

**改善案**:
- **ステッパー形式への変更**: 1画面に全フィールドを詰め込まず、「バッター選択 → ピッチャー選択（前回と同じなら自動） → 結果選択」の3ステップに分割
- **結果選択UIの改善**: ChoiceChipは機能するが、もっと大きなボタン（アイコン付き）で、片手で操作しやすいサイズに
  - 特に「ヒット」「アウト」「四死球」はアイコンを大きく、タップしやすく
- **連続入力の「もう1打席追加」ボタン**: 登録完了後に `context.pop()` せず、フォームリセットして次の入力に移行するオプション
- **最近使った選手の優先表示**: ドロップダウンの先頭に直近の試合で使った選手を表示

### 3.3 因縁一覧画面 (`matchup_list_page.dart`)

**現状**: シンプルなリスト表示。個人/チームのタブ切替。

**改善案**:
- **ソート機能**: 打率順、対戦数順、直近更新順で並び替え
- **フィルター機能**: 特定のチームの対戦相手のみ表示
- **ビジュアル強化**: 打率に応じてリストアイテムの背景色を変える（高打率 = 赤系、低打率 = 青系）
- **カード生成への直接リンク**: リストの各アイテムにシェアアイコンを追加し、詳細画面を経由せずにカード生成できるショートカット

### 3.4 因縁詳細画面 (`matchup_detail_page.dart`)

**現状**: ヘッダー（名前 VS 名前）+ 通算成績テーブル + カード生成ボタン。

**改善案**:
- **打席履歴タイムライン**: いつ何を打ったか（直近の打席結果を時系列で表示）
- **トレンドグラフ**: 打率の推移（試合ごとの通算打率の変化）を簡易グラフで
- **「因縁の歴史」セクション**: 初対戦日、最多HR試合、連続ヒット記録などのハイライト
- **直近N試合の成績**: 現在は通算のみ。直近3試合/5試合の成績を追加

### 3.5 因縁カード画面 (`card_preview_page.dart` + `rivalry_card_widget.dart`)

**現状**: カードプレビュー + 共有ボタンのみ。

**改善案**:
- **テンプレート選択**: 複数のカードデザインから選択可能に
- **編集機能**: タイトルやコメントをユーザーが編集できるように
- **保存機能**: 端末にカード画像を保存するボタンを追加（共有だけでなく保存も）
- **SNS別プレビュー**: 「Instagram用（正方形）」「Twitter用（16:9）」「LINE用（OGP）」

### 3.6 チーム詳細画面 (`team_detail_page.dart`)

**現状**: チーム情報 + 招待コード + メンバー + 試合一覧。

**改善案**:
- **招待コードの共有強化**: コピーだけでなく「LINEで招待」「QRコード表示」ボタンを追加
- **チーム成績サマリー**: 通算戦績（X勝Y敗Z分）をヘッダーカードに表示
- **メンバーの打撃成績**: メンバー一覧にチーム内での打率/HR数を表示

### 3.7 プロフィール画面 (`profile_page.dart` + `edit_profile_page.dart`)

**現状**: アイコン + 表示名 + 登録日 + ログアウト。情報が少ない。

**改善案**:
- **個人通算成績の表示**: 自分の打撃成績（通算打率、HR数など）
- **所属チーム一覧**: 複数チームに所属している場合の一覧表示
- **因縁ランキング**: 自分の因縁で最も打率が高い/低い相手TOP3
- **Claim管理**: 仮プレイヤーのClaim状態を確認・操作できるUI

## 4. コピーライティング改善

### 4.1 アプリ全体のトーン

**現状**: 「base_match」「草野球の因縁を記録しよう」-- 説明的だがやや無機質。

**改善方針**: 草野球プレイヤーの「仲間内のノリ」に寄り添う。真剣すぎず、ふざけすぎず、「ニヤリ」とさせる言葉選び。

### 4.2 画面別のテキスト改善

| 画面 | 現状 | 改善案 | 理由 |
|------|------|--------|------|
| ログイン | 「草野球の因縁を記録しよう」 | 「あいつとの因縁、忘れてないか？」 | 感情に訴える。「記録」は機能説明、「忘れてないか」は動機づけ |
| ホーム(空状態) | 「チームに参加していません」 | 「まずはチームを作ろう。因縁はそこから始まる。」 | ストーリー性を持たせる |
| 試合(空状態) | 「試合がありません」 | 「まだ因縁は生まれていない。最初の一戦を記録しよう。」 | 「因縁が生まれる」というワクワク感 |
| 因縁(空状態) | 「対戦成績がありません」 | 「まだ宿敵はいない。試合を記録して因縁を刻もう。」 | 「因縁を刻む」という世界観 |
| 打席入力完了 | (SnackBar等なし) | 「打席を記録した！因縁が深まる...」 | 入力のご褒美感 |
| 試合確定 | 「試合を確定すると...」 | 「この試合の因縁を確定する。覚悟はいいか？」 | ゲーム性を演出 |

### 4.3 因縁カードのコメントテンプレート拡充

**現状**（`_generateComment()` の5パターン）:

| 条件 | コメント |
|------|---------|
| HR >= 3 | 天敵認定 |
| avg >= .400 | 圧倒的な打棒! |
| avg >= .300 | 好相性の打者 |
| avg >= .200 | 互角の勝負 |
| avg < .200 | 苦手な投手 |

**改善案**: 複数の軸を組み合わせた30パターン以上

```
// 打率系
avg >= .500 && ab >= 10   → "完全支配"
avg >= .400               → "天敵認定"
avg >= .350 && hr >= 2    → "恐怖の強打者"
avg >= .300               → "相性抜群"
avg >= .250               → "五分の勝負"
avg >= .200               → "苦戦中..."
avg >= .100               → "難攻不落の壁"
avg < .100 && ab >= 10    → "天敵は向こうだった"

// 対戦数系
ab >= 30                  → "宿命のライバル"
ab >= 20                  → "因縁深まる"
ab >= 10                  → "顔なじみ"
```

## 5. グロース施策

### 5.1 ユーザー獲得

#### 因縁カード駆動の自然獲得（最重要）
- 因縁カードのSNS共有がそのままアプリの広告になる設計
- カードに小さくApp Storeリンク / QRコードを含める
- 共有テキストにハッシュタグとストアURLを自動挿入
- **目標**: カード共有 → 新規インストールのコンバージョン率 5%

#### 対戦相手への「因縁通知」
- 因縁カードを共有すると、受け取った相手（未登録の場合）にアプリへの招待メッセージが送れる
- 「{名前}さんがあなたとの因縁カードを作成しました。確認する？」
- 相手が登録すると自動Claimで因縁が「双方向」になる

#### 草野球コミュニティへのアプローチ
- 草野球のSNSコミュニティ（X/Twitter、Facebook草野球グループ）で因縁カードを投稿
- 草野球リーグ運営者への無料利用提供

### 5.2 継続利用

#### 試合リマインド
- 定期的な試合のある草野球チームに対し、「今週の試合を記録しましたか？」のリマインド
- 未入力の試合がある場合に通知

#### 因縁の「成長」演出
- 対戦数が増えるごとに因縁カードのランクが上がる
  - 3打席: 「因縁の始まり」
  - 10打席: 「宿敵認定」
  - 20打席: 「永遠のライバル」
  - 50打席: 「伝説の因縁」
- ランクアップ時に特別なカードデザインが解放される

#### 月間/シーズンレポート
- 月末に「今月の因縁レポート」を自動生成
  - 今月最も対戦した相手
  - 今月最も打率が変動した因縁
  - 新しく生まれた因縁
- シーズン終了時に年間ベスト因縁を表彰

#### ストリーク（連続記録）
- 「連続試合記録中」のバッジをプロフィールに表示
- 試合記録を連続で入力するとストリークが伸びる
- 「3試合連続入力で特別な因縁カードテーマ解放」のようなインセンティブ

### 5.3 マネタイズ（将来検討）

#### フリーミアムモデル
- **無料**: 基本機能（試合記録、因縁閲覧、カード生成（ウォーターマーク付き））
- **プレミアム（月額）**:
  - ウォーターマーク削除
  - 追加カードテーマ
  - 詳細スタッツ（OPS、長打率、出塁率など）
  - シーズンレポート
  - 過去の試合データエクスポート

#### チーム向けプラン
- チーム単位の月額課金
- 全メンバーのプレミアム機能解放
- チーム専用の統計ダッシュボード
