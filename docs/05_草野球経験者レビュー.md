# 草野球経験者レビュー

> レビュアー: 草野球歴15年、チームキャプテン兼スコア管理担当
> 対象: base_match アプリ全画面・全機能

---

## 総評

**結論: 仲間に勧めたい気持ちはあるが、現状では「面白そうだけど、入力がしんどい」と言われる可能性が高い。**

因縁カードのコンセプトは草野球プレイヤーにとって非常に刺さる。「あいつに何割打ってるか」「あのチームに勝ち越してるか」は、打ち上げの居酒屋で毎回のように出る話題だ。これをアプリで可視化してLINEに送れるのは、間違いなく盛り上がる。

ただし、現状のアプリでその体験にたどり着くまでのハードルが高い。草野球の「現場」を想像すると、以下の3点が大きな課題:

1. **試合後の入力が重い**: 着替えて居酒屋に向かう車の中で入力を始めるが、1打席ごとに選手選択をやり直す手間がある。ビール1杯目を飲み終わるまでに入力が終わらない。
2. **試合作成時に「草野球の事情」が考慮されていない**: 5イニング制やダブルヘッダーに対応していない。3イニングの練習試合もよくある。
3. **因縁カードのコメントが「お行儀が良すぎる」**: 草野球仲間は「天敵認定」より「もう敬遠しよう」「飲み会のネタ確定」のほうが笑える。カードを送って盛り上がるには、もう少し「仲間内のノリ」が必要。

## 試合入力フローの評価

### シミュレーション: 日曜の試合後、居酒屋に向かう車の中

1. アプリを開く。ホーム画面に「試合を記録」FABがあるのは良い。
2. チームが1つなら即試合作成画面へ。これは助かる。
3. **試合作成画面**: 日付と球場を入力、対戦チームを選ぶ。ここまでは問題ない。
   - **問題**: イニング数の設定がない。今日は5イニングだったのに、それを記録する場所がない。
   - **問題**: 今日はダブルヘッダーだった。第1試合と第2試合を区別できない。同じ日に同じ相手と2試合あると、どっちがどっちかわからなくなる。
4. **打席入力画面**: ここが最大のボトルネック。
   - イニングを手入力するのは良い（任意だし）。
   - バッターを選ぶ: チーム選択 → 選手選択。初回は選手が未登録なので「新規作成」→ チーム選択 → 名前入力。**これが全打席で必要になると辛い**。
   - ピッチャーは連続入力で保持されるようになっているのは評価できる（改善済み）。
   - **結果選択のChoiceChipは小さい**。車の中で揺れながら片手操作するには、もう少し大きなボタンが欲しい。
   - **犠打(バント)と犠飛がない**。草野球でもバントはするし、犠牲フライもよくある。結果詳細の「アウト」カテゴリに犠打・犠飛が欲しい。
   - **新規選手作成時のサジェストがない**: 相手ピッチャーの名前がわからない場合、「相手投手」「相手4番」のようなテンプレートがあると助かる。
5. 20打席入力して試合確定。連続入力モードがあるので以前よりは改善されている。
6. 確定後に因縁カード提案のボトムシートが出るのは良い改善。

### ストレスポイント

- **相手選手の名前がわからない問題**: 草野球では相手チームの選手名を全員知っていることは稀。「あのサウスポー」「ショートの人」程度の認識。新規作成時に「相手投手」「相手捕手」等のテンプレートが欲しい。
- **同一試合で投手交代がある場合**: 5イニングの試合でも投手交代は珍しくない。1-3回が田中で、4-5回が鈴木、のような場合にピッチャーを切り替える操作がやや面倒。
- **打席結果に犠打・犠飛がない**: 「アウト → その他」で代用はできるが、統計的に正確でなくなる。犠打・犠飛は打数に含まれないため、打率計算にも影響する（本来は打数から除外すべき）。

## 因縁カードの評価

### LINEグループに送って盛り上がるか?

**デザインは十分にかっこいい**。ダークグリーンのグラデーション背景にゴールドの枠線、「VS」の赤い斜線アクセント。これは草野球仲間に見せたら「おお!」となるクオリティ。

**しかし、コメントが「真面目すぎる」**。

現状のコメントパターンは「天敵認定」「相性抜群」「五分の勝負」などで、スポーツニュースのような硬さがある。草野球の仲間内では、もっとくだけた表現のほうが盛り上がる:
- 「飲み会のネタ確定」 → 結果を見ながら居酒屋で盛り上がるイメージが湧く
- 「次の試合が楽しみだ」 → 次の対戦への期待感
- 「リベンジ必至」 → 負けている側の闘志
- 「助っ人コールが必要」 → 草野球あるある（強い相手にはメンバーを補強する）
- 「打ち上げで自慢できる」 → 勝っている側の得意げな感じ
- 「次は絶対打つ」 → 打てていないバッターの宣言
- 「投げるのが怖い打者」 → ピッチャー目線の恐怖
- 「もう敬遠しよう」 → 手が付けられない打者への白旗

これらのコメントがカードに入っていれば、LINEグループに送ったときに笑いと議論が起きる。

### カードに足りないもの

- **「次の試合で使える」感**: カードを送りつけることで、次の試合への煽りになる要素
- **チーム対戦カードの盛り上がり**: 個人因縁は面白いが、チーム対戦の「お得意様」「苦手チーム」も仲間内では重要な話題

## 「草野球あるある」への対応状況

| あるある | 対応状況 | 評価 |
|----------|----------|------|
| 5イニング/3イニング制 | 未対応。イニング数の概念がない | 要改善 |
| ダブルヘッダー | 未対応。同日同一相手の複数試合を区別できない | 要改善 |
| 助っ人（1試合だけ参加する人） | 仮プレイヤーとして登録可能。ただし名前を毎回入力する手間あり | 部分的に対応 |
| 相手選手の名前がわからない | 仮プレイヤーで「相手投手」等と入力可能だが、テンプレートなし | 要改善 |
| 犠打・犠飛 | 結果詳細にない。「アウト→その他」で代用 | 要改善 |
| 選手が複数チームに所属 | 同一user_idで複数teamのplayerを持てるので対応可能 | 対応済み |
| 雨天中止 | 試合削除機能がないが、下書きのまま放置は可能 | 部分的 |
| 投手交代 | 打席ごとにピッチャーを変更可能。連続入力でピッチャー保持されるので交代時に切り替えれば良い | 対応済み |

## 改善提案（優先度付き）

### 優先度: 高（これがあれば絶対使う）

1. **試合作成時のイニング数選択**: 3/5/7/9イニングを選べるようにする。草野球は7イニングが標準だが、時間や人数の都合で5イニングや3イニングも多い。
2. **ダブルヘッダー対応**: 同日に複数試合ある場合の試合番号（第1試合/第2試合）を設定できるようにする。
3. **犠打・犠飛の追加**: 打席結果の「アウト」カテゴリに犠打（犠牲バント）と犠飛（犠牲フライ）を追加。将来的には打率計算時に打数から除外する処理も検討。
4. **仮プレイヤー作成時のテンプレート**: 「相手投手」「相手捕手」「相手4番」等の表示名テンプレートをサジェスト。
5. **因縁カードに草野球ノリのコメント追加**: 「飲み会のネタ確定」「もう敬遠しよう」等、仲間内で盛り上がるコメントパターン。

### 優先度: 中（あると便利）

6. **試合詳細のスコア表示強化**: ホームvsアウェイのスコアをもっと大きく見やすく。
7. **イニング別の打席結果表示**: 「1回: 田中→ゴロ、鈴木→単打、佐藤→三振」のように時系列で見れると、試合を振り返りやすい。
8. **個人通算成績の表示**: プロフィールに自分の通算打率、HR数を表示。
9. **試合メモ機能**: 「雨で5回コールド」「助っ人3人」のようなメモを残せると良い。

### 優先度: 低（あると嬉しい）

10. **試合結果の手動入力**: 現在はRBIからスコアを自動計算しているが、実際のスコアを手動で入力できるオプション。エラーや暴投での得点が反映されないため。
11. **打順表示**: 打席結果のリストに「1番」「2番」等の打順を表示。
12. **天候の記録**: 晴れ/曇り/小雨等。意外と「あの雨の試合で...」という思い出に紐づく。

## 実装した改善内容

以下の改善をコードに直接実装した:

### a) 試合作成画面 (create_game_page.dart)
- イニング数の選択を追加（3/5/7/9イニング、デフォルト7）
- ダブルヘッダー対応（試合番号: 第1試合/第2試合の選択）

### b) 因縁カードのコメント拡充 (rivalry_card_widget.dart)
- 草野球プレイヤーが仲間内で盛り上がるサブコメントを追加
- メインコメントに加えて「飲み会のネタ確定」「もう敬遠しよう」等のサブコメントを表示

### c) 打席入力画面 (plate_appearance_input_page.dart)
- 犠打（犠牲バント）と犠飛（犠牲フライ）の結果詳細を追加
- 仮プレイヤー作成時のテンプレート名サジェスト（「相手投手」「相手4番」等）

### d) ゲーム詳細画面 (game_detail_page.dart)
- イニングごとの打席結果をグルーピングして見やすく表示
- スコア表示を改善（チーム名とスコアの大きな表示）
