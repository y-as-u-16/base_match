# MVP1 ローカル特化版 設計書

## 1. 目的

MVP1 では、Supabase、ログイン、課金、チーム共有、因縁カード共有をいったん本線から外し、端末内だけで草野球の記録と成績確認を完結させる。

この段階で作る価値は次の 3 つに絞る。

- 試合を記録できる
- 打席やピッチング成績を入力できる
- 入力した成績を見られる

MVP1 のゴールは「多機能な野球 SNS」ではなく、「ログインなしで迷わず使える草野球のローカル記録アプリ」を完成させること。

## 2. 基本方針

### 2.1 作るもの

- 未ログインで使えるホーム
- 試合の作成、一覧、詳細
- 打席成績の入力
- ピッチング成績の入力
- 打撃成績の一覧、詳細
- ピッチング成績の一覧、詳細
- 端末内への永続保存

### 2.2 作らないもの

- Supabase 接続
- ログイン、新規登録
- プロフィール
- チーム作成、チーム参加
- 招待コード
- 因縁一覧、因縁詳細
- 因縁カード生成、共有
- 課金導線
- Cloud 同期
- Claim、名寄せ

既存コードとして残す場合でも、MVP1 の画面導線、ボトムナビ、Provider からは外す。

## 3. ボトムナビ構成

MVP1 のボトムナビは 3 つに絞る。

| タブ | 役割 | ルート |
|---|---|---|
| ホーム | 入口、主要 CTA、直近状況 | `/` |
| 記録 | 試合、打席、ピッチング成績の入力 | `/games` |
| 成績 | 打撃成績、ピッチング成績の確認 | `/stats` |

現在の `チーム`、`因縁`、`プロフィール` タブは削除する。

ホームに何を表示するかは未定とし、初期実装では最小限の CTA と空状態だけでよい。

## 4. 画面一覧

| 画面 | ルート | 説明 |
|---|---|---|
| ホーム | `/` | アプリ入口。表示内容は後で詰める |
| 試合一覧 | `/games` | ローカル保存された試合一覧 |
| 試合作成 | `/games/create` | 日付、自チーム、相手チーム名、スコアなどを入力 |
| 試合詳細 | `/games/:gameId` | 試合情報、打席、ピッチング成績を表示 |
| 打席入力 | `/games/:gameId/plate-appearances/new` | 1 打席分の打撃結果を入力 |
| ピッチング入力 | `/games/:gameId/pitching/new` | 投手成績を入力 |
| 成績トップ | `/stats` | 打撃/投手成績への入口 |
| 打撃成績 | `/stats` | 自チームの打撃成績 |
| 投手成績 | `/stats` | 自チームのピッチング成績 |

設定画面は MVP1 の必須範囲には含めない。ローカルデータ削除などが必要になった時点で追加する。

## 5. 各画面の設計

### 5.1 ホーム

ホームの表示内容は未定とする。

初期実装で置いてよいもの:

- `試合を記録する` CTA
- `成績を見る` CTA
- 直近の試合がない場合の空状態
- ローカル保存であることを示す短い補足

表示しないもの:

- ログイン CTA
- チーム作成 CTA
- 因縁カード CTA
- プロフィール導線
- 課金導線

### 5.2 記録タブ

記録タブは、試合を起点に入力する。

主な役割:

- 試合一覧を表示する
- 新しい試合を作成する
- 既存の試合詳細へ進む
- 試合詳細から打席入力、ピッチング入力へ進む

MVP1 では共有やメンバー管理を持つチーム機能は作らない。
ただし、自チームは毎回入力せずに済むよう、端末内の `local_my_teams` で少数の自チーム候補を管理する。
相手チームは対戦ごとに変わる想定のため、試合ごとの文字列として保存する。

### 5.3 試合作成

入力項目:

- 試合日
- 自チーム選択
- 相手チーム名
- 自チーム得点
- 相手チーム得点
- 球場
- イニング数

不要にするもの:

- 共有チームの `teamId` 必須
- 共有チームの選択
- 対戦相手チームの新規作成
- 招待コード
- チームメンバー管理

### 5.4 試合詳細

表示するもの:

- 試合日
- 自チーム名
- 相手チーム名
- 球場
- 状態: 下書き / 確定
- 打席一覧
- ピッチング成績一覧
- `打席を追加`
- `ピッチング成績を追加`
- `試合を確定`

MVP1 の試合確定は「入力完了マーク」に近い扱いにする。確定後の編集可否は実装時に判断する。

### 5.5 打席入力

MVP1 の打席入力は、個人成績として集計できるように打者名を保存する。

入力項目:

- イニング
- 打者名
- 結果タイプ
  - ヒット
  - アウト
  - 四死球
  - エラー
- 結果詳細
  - 単打、二塁打、三塁打、本塁打
  - 三振、ゴロ、フライ、ライナー、併殺、犠打、犠飛、その他
  - 四球、死球
  - エラー
- 打点

MVP1 では選手 master は作らず、打席ごとに打者名を文字列で保存する。表記ゆれは残るが、まずは個人成績を軽く入力できることを優先する。

### 5.6 ピッチング入力

ピッチング入力は、投手成績を手入力で残す。

入力項目:

- 投球回
- 投手名
- 失点
- 自責点
- 被安打
- 与四死球
- 奪三振
- 被本塁打

MVP1 では打席入力から投手成績を完全自動計算しない。まずは手入力で投手成績を保存できる状態を優先する。

### 5.7 成績タブ

成績タブは、打撃成績とピッチング成績を切り替えて確認できる画面にする。

表示するもの:

- 打撃成績一覧への導線
- ピッチング成績一覧への導線
- データがない場合の空状態

### 5.8 打撃成績

個人別に表示するもの:

- 打席数
- 打数
- 安打
- 本塁打
- 四死球
- 三振
- 打率

詳細で表示するもの:

- 個人別の通算打撃成績
- 最近の打席履歴

### 5.9 ピッチング成績

個人別に表示するもの:

- 登板数
- 投球回
- 失点
- 自責点
- 被安打
- 与四死球
- 奪三振
- 防御率

詳細で表示するもの:

- 個人別の通算投手成績
- 試合別の投球履歴

## 6. データ設計

### 6.1 保存先

MVP1 は端末内 SQLite に保存する。

- DB ファイル: `base_match_local.sqlite`
- 保存場所: アプリの documents directory
- 実装候補: Drift + SQLite

### 6.2 テーブル

MVP1 では次のテーブルから開始する。

- `local_my_teams`
- `local_games`
- `local_plate_appearances`
- `local_pitching_appearances`

### 6.3 local_my_teams

| カラム | 説明 |
|---|---|
| id | UUID |
| name | 自チーム名 |
| color_key | UI でチームを見分けるための色キー |
| is_default | 試合作成時に初期選択するチームかどうか |
| display_order | チーム選択肢での並び順 |
| archived_at | 使わなくなったチームの非表示日時 |
| created_at | 作成日時 |
| updated_at | 更新日時 |

### 6.4 local_games

| カラム | 説明 |
|---|---|
| id | UUID |
| date | 試合日 |
| location | 球場 |
| my_team_id | `local_my_teams.id` |
| away_team_name | 相手チーム名 |
| home_score | 自チームスコア |
| away_score | 相手チームスコア |
| status | draft / final |
| innings | イニング数 |
| created_at | 作成日時 |
| updated_at | 更新日時 |

### 6.5 local_plate_appearances

| カラム | 説明 |
|---|---|
| id | UUID |
| game_id | 試合 ID |
| batter_name | 打者名 |
| inning | イニング |
| result_type | hit / out / walk / error |
| result_detail | single / double / triple / hr など |
| rbi | 打点 |
| created_at | 作成日時 |

### 6.6 local_pitching_appearances

| カラム | 説明 |
|---|---|
| id | UUID |
| game_id | 試合 ID |
| pitcher_name | 投手名 |
| outs_pitched | 投球アウト数。1回 = 3 |
| runs | 失点 |
| earned_runs | 自責点 |
| hits_allowed | 被安打 |
| walks | 与四死球 |
| strikeouts | 奪三振 |
| home_runs_allowed | 被本塁打 |
| created_at | 作成日時 |

投球回は小数ではなくアウト数で保存する。表示時に `1回1/3` のように変換する。

## 7. 集計仕様

### 7.1 打撃成績

打撃成績は `batter_name` ごとに集計する。

| 指標 | 計算 |
|---|---|
| PA | 全打席数 |
| AB | hit / out / error |
| H | result_type = hit |
| 2B | result_detail = double |
| 3B | result_detail = triple |
| HR | result_detail = hr |
| BB/HBP | result_type = walk |
| SO | result_detail = k |
| RBI | rbi の合計 |
| AVG | H / AB |
| OBP 簡易 | (H + BB/HBP) / PA |

`AB == 0` の場合、打率は `0.000` 表示にする。

### 7.2 ピッチング成績

ピッチング成績は `pitcher_name` ごとに集計する。

| 指標 | 計算 |
|---|---|
| G | 登板数 |
| IP | outs_pitched / 3 |
| R | runs の合計 |
| ER | earned_runs の合計 |
| H | hits_allowed の合計 |
| BB/HBP | walks の合計 |
| SO | strikeouts の合計 |
| HR | home_runs_allowed の合計 |
| ERA | ER * 9 / IP |

`outs_pitched == 0` の場合、防御率は `-.--` または `0.00` のどちらで表示するか実装時に決める。

### 7.3 チームスコア

MVP1 のチームスコアは試合作成時に手入力する。

- `home_score` は自チーム得点として入力する
- `away_score` は相手チーム得点として入力する
- 打席入力の `rbi` やピッチング入力の `runs` から試合スコアを自動更新しない

打点や失点は個人成績の入力値として扱い、試合結果のスコアとは独立させる。

## 8. 削除・隔離する機能

### 8.1 画面導線から消すもの

- `/login`
- `/sign-up`
- `/teams`
- `/teams/create`
- `/teams/join`
- `/teams/:teamId`
- `/matchups`
- `/matchups/:matchupType/:id1/:id2`
- `/matchups/:matchupType/:id1/:id2/card`
- `/profile`
- `/profile/edit`

MVP1 ではロック付きプレビューも原則出さない。使えない機能を見せるより、記録と成績に集中させる。

### 8.2 コードから削除するもの

MVP1 方針で不要になったものは、通常導線から外すだけでなくコードから削除する。

- Supabase 接続 Provider
- Supabase Repository 実装
- Email/Password ログイン画面
- Email/Password 新規登録画面
- Auth domain/entity
- Team domain/entity
- プロフィール編集画面
- 因縁カード共有導線
- 招待コード入力画面

将来ログインを戻す場合は Google / Apple 前提で作り直す。

## 9. アーキテクチャ方針

### 9.1 Local only

MVP1 の Provider は Local Repository だけを返す。

避けること:

- Supabase Provider を残す
- Auth ユーザー必須の Repository を残す
- 未提供機能を Overlay で隠して残す
- BottomNav に未提供機能を残す

### 9.2 推奨ディレクトリ

```text
lib/
  core/
    local_db/
    routing/
    theme/
    widgets/
  features/
    home/
      presentation/
    games/
      data/
      domain/
      presentation/
    stats/
      domain/
      presentation/
```

MVP1 では `auth`、`teams`、`profile`、`matchups`、`share` を削除する。

### 9.3 責務分割

| 責務 | 置き場所 |
|---|---|
| SQLite テーブル定義 | `core/local_db` |
| 自チーム候補 CRUD | `features/games/data` |
| 試合 CRUD | `features/games/data` |
| 打席入力 CRUD | `features/games/data` |
| ピッチング入力 CRUD | `features/games/data` |
| 打撃成績計算 | `features/stats/domain` |
| ピッチング成績計算 | `features/stats/domain` |
| 成績表示 Provider | `features/stats/presentation` |
| 画面 | 各 feature の `presentation/pages` |

成績計算は Repository に閉じ込めず、純粋 Dart の calculator にする。これにより unit test を書きやすくする。

## 10. 実装順

### Step 1. ルートと BottomNav を MVP1 専用にする

- BottomNav を `ホーム / 記録 / 成績` に変更
- Auth、Teams、Profile、Matchups、Share のルートを削除する
- `/games/create` の共有チーム `teamId` 必須をやめる

### Step 2. Home を最小化する

- チーム作成 CTA を削除
- 因縁カード系の表示を削除
- Supabase/currentUser 依存 Provider を読まない

### Step 3. ローカル DB を追加する

- Drift + SQLite を追加
- `local_my_teams`
- `local_games`
- `local_plate_appearances`
- `local_pitching_appearances`

### Step 4. 記録入力をローカル化する

- 自チームを `local_my_teams` から選択できるようにする
- 相手チームはチーム名文字列ベースにする
- 打席入力を打者名入力ベースにする
- ピッチング入力画面を追加する
- ピッチング入力を投手名入力ベースにする
- 入力後に試合詳細へ反映する

### Step 5. 成績画面を追加する

- 打撃成績一覧
- 打撃成績詳細
- ピッチング成績一覧
- ピッチング成績詳細

### Step 6. 不要機能を削除または隔離する

- Auth/Teams/Profile/Matchups/Share のコードを削除する
- MVP1 で参照されなくなった Provider を整理する
- Supabase 初期化を MVP1 起動の必須条件から外す

### Step 7. テストを追加する

- 試合作成の unit/widget test
- 打席入力の unit/widget test
- ピッチング入力の unit/widget test
- 打撃成績 calculator の unit test
- ピッチング成績 calculator の unit test

## 11. MVP1 完了条件

- Supabase 未設定で起動できる
- ログイン画面が出ない
- BottomNav が `ホーム / 記録 / 成績` になっている
- ホームから試合作成へ進める
- 試合作成後に打席入力へ進める
- 試合作成後にピッチング入力へ進める
- 打席入力が名前入力だけで完了できる
- ピッチング成績を保存できる
- 入力結果が試合詳細に出る
- 入力結果が成績画面に反映される
- アプリ再起動後もデータが残る
- MVP1 の通常導線にログイン、チーム、プロフィール、因縁、課金、共有が出ない
- `flutter analyze` が通る
- `flutter test` が通る
