# 草野球対戦履歴アプリ 基本設計書

**目的**：草野球における バッター×ピッチャー および チーム×チーム の対戦履歴を蓄積し、因縁（相性）を「見せたくなるカード」として共有できるプロダクトを実現する。

---

## 1. プロダクト概要

### 1.1 コンセプト

- 草野球版・対戦履歴メディア
- 片側（自分だけ）の入力でも成立し、相手が後から参加して「名寄せ／本人化」されることで価値が増す（ネットワーク効果）

### 1.2 提供価値

- **個人**：
  - 自分の打撃/投球成績の整理
  - 特定投手/打者との相性が見える
- **チーム**：
  - チームの対戦史（宿敵、通算戦績、最近の流れ）
- **共有**：
  - 因縁カード（SNS/LINE向け画像）で拡散しやすい

### 1.3 主要KPI（目安）

- 初回価値到達：初回起動〜「打席入力 / 個人成績表示」まで 3分以内
- 継続：月1回以上の試合記録率
- 招待：カード共有→相手の新規登録率

### 1.4 アクセスモデル

- **未ログイン**：
  - 端末内だけで試合作成、打席入力、個人成績表示まで利用可能
  - データ保存先はローカルのみ。端末変更・削除時の復元はできない
- **ログイン済み（無料）**：
  - ローカル記録をクラウドへ保存・同期できる
  - 複数端末での利用、端末変更時の復元、Claim を利用できる
- **課金ユーザー**：
  - チーム作成/参加、招待コード、チーム共有を利用できる
  - 因縁機能（個人/チーム）の生成・共有を利用できる
  - 共有された因縁カードの閲覧自体は無料

### 1.5 段階リリース方針

- **Phase 1**：
  - 未ログインのローカル記録体験だけを先行リリースする
  - この段階では Supabase を利用しない
- **Phase 2**：
  - Google / Apple ログイン、クラウド同期、ローカル記録の移行を追加する
- **Phase 3**：
  - 課金、チーム機能、因縁生成・共有を追加する

## 4. 機能要件

### 4.1 認証・アカウント

- **ローカルモード**
  - 未ログインでも端末内だけで利用開始できる
  - 対象は試合作成、打席入力、個人成績表示まで
- **Supabase Auth**
  - Google / Apple（Phase 2）
- **ユーザープロフィール**
  - 表示名（必須）
  - 野球利き腕、ポジション（任意）
  - アイコン（任意）
- **クラウド移行**
  - ログイン時にローカル記録をクラウドへ移行できる
  - 移行前に件数と対象データを確認できる

### 4.2 チーム

- **利用条件**
  - チーム機能は課金ユーザー向け
- **チーム作成**
  - チーム名（必須）
  - 都道府県/活動エリア（任意）
  - アイコン（任意）
- **招待コード発行**
- **参加申請 or 即時参加**（MVPは即時）
- **メンバー管理（ロール）**
  - Owner（作成者）
  - Admin（記録編集可）
  - Member（閲覧＋共有）

### 4.3 試合

- **試合作成**
  - 日時
  - 球場（任意）
  - 自チーム（必須）
  - 相手チーム（選択：登録済 or 仮チーム作成）
- **試合状態**
  - Draft（作成中）
  - Final（確定）

### 4.4 打席イベント（MVP簡易）

- 1打席 = 1イベントとして入力
- **入力項目**
  - イニング（任意）
  - バッター（登録済 or 仮プレイヤー）
  - ピッチャー（登録済 or 仮プレイヤー）
  - 結果（必須）
    - OUT（ゴロ/フライ/三振/併殺/…）
    - HIT（単打/二塁打/三塁打/本塁打）
    - WALK（四球/死球）
    - ERROR
  - 打点（任意）
- **集計対象（MVP）**
  - 打数、安打、HR、四死球、三振、出塁率（簡易）

### 4.5 対戦成績（因縁）

- **利用条件**
  - 因縁の生成・共有は課金ユーザー向け
  - 共有された因縁の閲覧は無料
- **バッター×ピッチャー**
  - 通算：打数/安打/HR/四死球/三振
  - 直近N試合（N=3/5）
  - 相性指標（例：打率、OPS簡易）
- **チーム×チーム**
  - 通算：勝敗/得点/失点
  - 直近N試合

### 4.6 因縁カード生成・共有

- **利用条件**
  - 因縁カードの生成・共有は課金ユーザー向け
  - 閲覧URL/共有先からの閲覧は無料
- **カード種類**
  - 個人因縁（Batter vs Pitcher）
  - チーム因縁（Team vs Team）
- **生成内容**
  - タイトル（テンプレ＋AIオプション）
  - 通算成績
  - 直近成績
  - 一言コメント（テンプレ or AI）
- **出力**
  - 画像（端末側でレンダリング→共有）
  - Webリンク（将来：Dynamic Links/Universal Links）

### 4.7 仮プレイヤー／名寄せ（Claim）

- **仮プレイヤー**
  - 未登録の相手選手を入力するための器
  - 表示名のみ必須（「◯◯クラブ 右腕エース」等）
- **Claim**
  - 仮プレイヤーに「本人」が紐づく
  - 以後、対戦履歴が本人プロフィールに統合表示

### 4.8 修正・異議申し立て（MVP最小）

- 試合の編集権限：Owner/Admin
- 申し立て：Memberが「修正依頼」送信（通知は後続）
- MVPは「編集履歴（監査ログ）」のみ実装し、ワークフローは簡易

### 4.9 課金/閲覧権限

- **無料（未ログイン）**
  - ローカル記録、打席入力、個人成績表示
- **無料（ログイン済み）**
  - クラウド同期、データ復元、Claim
- **有料**
  - チーム機能
  - 因縁の生成・共有
- **閲覧無料**
  - 共有された因縁カード

## 7. Supabase 設計

### 7.1 Supabase Auth

- 認証は Phase 2 以降で Supabase Auth を利用する
- JWT は Supabase が発行し、RLS ポリシーで `auth.uid()` を利用
- ソーシャルログインは Google / Apple を採用する
- 未ログイン時のデータは Supabase に保存せず、端末内ストレージに保持する

### 7.2 Supabase Storage（将来）

- チームアイコン、プロフィール画像の保存
- 因縁カード画像のキャッシュ（オプション）

### 7.3 Supabase Edge Functions

- 因縁カードのAIテキスト生成（LLM呼び出し）
- 外部API連携

### 7.4 ディープリンク（将来）

- iOS: Universal Links
- Android: App Links
- 因縁カード/試合ページの共有URLに対応
- 公開閲覧用の共有データは、認証必須データとは別の公開用投影を持つ

### 7.5 プッシュ通知（将来）

- Supabase には組み込みのプッシュ通知機能がないため、以下の選択肢を検討
  - OneSignal
  - Pusher
  - 自前実装（APNs / FCM直接）
- 用途：Claim通知、修正依頼

## 10. 名寄せ（Claim）詳細

### 10.1 Claim対象

- `players.user_id is null` の仮プレイヤー

### 10.2 Claim手段（MVP案）

- 自分の名前が含まれるプレイヤー候補一覧から選ぶ
- 該当試合に参加したユーザーのみ Claim 可能（RLSで制限）

### 10.3 Claim後の表示

- `players.user_id` に紐づけ
- 以後、プロフィール画面で「あなたの対戦史」として表示

## 11. 非機能要件

### 11.1 パフォーマンス

- 試合詳細：イベント100件程度を想定
- 因縁一覧：ページング必須
- 集計ビューはDB側で実施

### 11.2 データ整合性

- 打席イベントが一次データ
- ゲーム確定（final）後の編集は Admin のみに制限（将来：ロック＋申請）

### 11.3 セキュリティ

- RLS適用
- 共有リンクは閲覧のみ
- 個人情報は最小（メール/表示名のみ）
- 未ログイン利用時のデータは端末内保存とし、クラウド送信しない

### 11.4 障害時

- 監査ログ（audit_logs）
- リカバリ：plate_appearances を正として再集計可能

### 12.1 マイルストーン

- **M1**：ローカル記録 + 打席入力 + 個人成績表示

## 13. 仕様メモ（重要な判断）

### 13.1 入力設計の割り切り

- MVPでは「打席イベント」だけを正確に。
- 守備や走塁などは捨てる。

### 13.2 初期ネットワーク問題の回避

- 相手未登録でも仮で入力可能。
- 相手が後から Claim できる。

### 13.3 バズ導線

- 因縁カードは 共有前提UI（1タップで画像生成→共有）。
- 小さな透かし（アプリ名）を入れる（課金でOFF）。

### 13.4 アクセス境界

- コア記録体験は未ログインでも開始できる
- クラウド保存はログイン必須
- チーム共有と因縁機能は課金対象
- 共有済み因縁の閲覧は無料
- Phase 1 ではクラウド保存機能自体を提供しない
