# デバッグ

$ARGUMENTS に関するバグを調査・修正してください。

## 調査手順

1. **エラーの再現**: エラーメッセージやスタックトレースから発生箇所を特定
2. **原因の特定**: 関連コードを読み、根本原因を特定
3. **影響範囲の確認**: 同じ問題が他の箇所にもないか確認
4. **修正の実施**: 最小限の変更で修正
5. **検証**: 修正後にテストを実行して確認

## 調査のコツ

- スタックトレースは下から上へ読む（直接の原因が上にある）
- `git log` / `git blame` で「いつから壊れたか」を特定
- 関連するログ出力を確認
- 同じエラーメッセージでGrepして類似箇所を探す

## 報告形式

```
## 原因
（なぜこのバグが発生したか）

## 修正内容
（何をどう変更したか）

## 影響範囲
（この修正が他に影響する箇所）

## 再発防止
（同じバグを防ぐための提案）
```
