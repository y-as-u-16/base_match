# リファクタリング

$ARGUMENTS をリファクタリングしてください。

## 手順

1. 対象コードを読み取り、現状の問題点を分析
2. リファクタリング計画を提示（変更前に必ずユーザーに確認）
3. 承認後、段階的に変更を実施
4. 既存テストが通ることを確認

## リファクタリング観点

- **関数の分割**: 長すぎる関数を責務ごとに分割
- **重複の排除**: DRY原則に基づく共通化
- **命名の改善**: 意図が明確な変数名・関数名
- **複雑度の低減**: ネストの深いif文、複雑な条件式の簡素化
- **型安全性**: 適切な型定義の追加
- **デザインパターン**: 適切なパターンの適用

## ルール

- 動作を変えない（振る舞いの保持）
- 一度に大きく変えすぎない
- 変更前後でテストが通ることを確認
- 不要な抽象化・過度な設計は避ける
