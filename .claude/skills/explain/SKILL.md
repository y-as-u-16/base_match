# コード解説

$ARGUMENTS を初心者にも分かりやすく解説してください。

## 解説の方針

1. **全体像を先に**: まず「何のためのコードか」を1-2文で説明
2. **段階的に詳細化**: 概要 → 主要な関数/クラス → 重要な処理の流れ
3. **専門用語は補足**: 初めて出てくる概念には簡単な説明を添える
4. **具体例で説明**: 抽象的な処理はデータの具体例を使って説明
5. **依存関係**: このコードが他のどの部分と連携しているかを示す

## 出力形式

```
## 概要
（このコードは何をするものか）

## 処理の流れ
（ステップバイステップ）

## 重要なポイント
（理解しておくべきこと）

## 関連ファイル
（このコードと連携している部分）
```
