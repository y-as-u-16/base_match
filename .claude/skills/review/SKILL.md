# コードレビュー

対象のコード変更をレビューしてください。

## 引数の解釈

- 引数なし: `git diff` で未コミットの変更をレビュー
- ファイルパス指定: そのファイルをレビュー
- ブランチ名指定: `git diff <branch>...HEAD` でレビュー

## レビュー観点

1. **バグ・論理エラー**: off-by-one、null参照、競合状態など
2. **セキュリティ**: インジェクション、認証漏れ、機密情報の露出
3. **可読性**: 命名、関数の長さ、コメントの適切さ
4. **パフォーマンス**: N+1クエリ、不要なループ、メモリリーク
5. **テスト**: テストの有無、カバレッジ、エッジケース

## 出力形式

問題を以下の優先度で分類して報告:

- **CRITICAL**: 必ず修正すべき問題（バグ、セキュリティ）
- **WARNING**: 修正を推奨する問題
- **SUGGESTION**: 改善提案

各問題には「ファイル:行番号」と具体的な修正案を含めてください。
最後にレビュー全体のサマリーを1-2文で述べてください。
